Intel 8086
Состоит из двух блоков.
Блок исполнения  	Блок интерфейса шины
Интерфейс - совокупность рельс которые обеспечивают сопряжение устр-в и программных модулей, как на физиическом так и на логическом уровне
В состав исполнительного блока входят: АЛУ(ALU), устройство управления(CU), 10 регистров
Устройства блока обеспечивают обработку комманд, выполнение логических арифметич. операциц
Три части БЛОКА Интерфеса шин:
Устройство управления шинами, устройство управления очередью комманд и сегментные регистры
Они предназначены: 
1. управление обменом данными с исп блоком, памятью и external units ввода вывода
2. адрессация памяти
3. выборка комманд(Instraction queoe)

Процессор состоит с точки зр-ия программиста:
8 регистров общего назначения: AX(AH AL) BX(BH BL) DX(DH DL) CX(CH CL)
4 сегментных регистра
регистра адресса комманд (IP)
регистр флагов (flags/...)

	Программно доступный
В состав 8086 входят 14-ть 16-тибитных регистров, которые исп-ся для управления/выполнения комманд, адрессация операндов


Все регистры:

1.РОН(общего назначения) (16-ти AX BX CX DX). Ф-ия, которая определяется регистром, опр-ся команлдами в которых он используется. С каждым регистром связано стандартное его применение
AX - accumulater аккумулятор (автоматически применяется в операциях умножения, деления и комманд ввод вывода (in out)
BX - base Р базы (*может содержать адрессаты эл-ов оперативной памяти, которые по умолчанию представляют собой СМЕЩЕНИЯ в сегменте данных)
CX - Counter счетчика (используется в различных операций, для хранения количества повторений. Например, в организации строковой комманды)
DX - Data Р данных (применяется в операциях умножения и деления в качестве расширителя)
Эти 4 регистра позволяют независимо обращаться и к старшей и младшей половинам. Соответсв. подрегистры являются (AH(high) AL(Low)).



2. Адрессные регистры - 4 Р.
BX -
SI - sourse index  также как и BX (*может содержать адрессаты эл-ов оперативной памяти, которые по умолчанию представляют собой СМЕЩЕНИЯ в сегменте данных)
DI - destination index также (*может содержать адрессаты эл-ов оперативной памяти, которые по умолчанию представляют собой СМЕЩЕНИЯ в сегменте данных)
BP - base point (-//- для сегмента стека)
При выполнении стр комманд SI содержит смещения источника в сегменте данных, а DI смещения в системе дополнительных.
Все перечисленные регистры можно использовать и для хранения данных, если они не используются по своему применению.

SP - Stack pointer Указатель стека (для записи данных в стек, чтения их из стека) (фактически содержит смещения вершины стека)(значения этого регистра автоматически изменяются коммандами PUSH. POP. PUSHF. POPF. CALL. RET. INT

(EIP)IP - особый регистр Instruction pointer Указатель инструкций(адресса команд) (всегда содержит смещение кода следующих выполняемых комманд)

Как только некоторая комманда выполняется - значение IP увеличивается на ее длинну, адресс будет - след комманды
Обычно комманды дополняются в последовательном виде, так как размещены в программе.

Нарушают последовательность команды перехода (JXX), комманда вызова процедуры (CALL), обработчика прерывания (INT), RET, IRET. (*LOOP - оператор цикла)
Косвенно загрузить в IP значения могут данные комманды



4. Сегментные регистры
CS - регистр сегмента кода (code) Он содержит адрес сегмента кода, к которому в данный момент имеет доступ микропроцессор.
DS - data segment Он содержит адрес сегмента данных текущей программы.
SS - stack segment (определяет адресс сегмента, в котором - стек)
ES(GS, FS) - (extension data segment registers) регистр сегмента доп данных (определяет смещение доп данныХ)

SP:BP (по умолчанию работа стека)
по умолчанию BX:SI:DI - определяют смещение данных
физ. адресс комманды в памяти определяет пара регистров CS:IP

5. Регистры флагов (регистры состояния) (регистры признаков) 
Включает в себя биты, каждый из которых устанавливается в 0/1 при опр.усл.

1. CF - флаг переноса (Carry flag). Принуд. Установка - STS(Set 1) CTS(Clear 0)
2. PF - (Parity) флаг четности (паритета)
3. AF - вспомогательный флаг переноса (Auxiliary carry Flag)
4. ZF - Zero flag (1 - нуль. 0 - ненуль)
5. SF - устанавливается в единицу, если в старшем бите содержится 1 (Sign flag регистр знака)
6. TF - флаг трассировки. Если T == 1, то по завершению каждой комманды возникает прерывание МикроПроцессора ()
